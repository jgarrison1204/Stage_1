Key Concepts: The Web, HTML and CSS

Lesson 1: The Basics of the Web

How the web works:

There are three major components that makes the web work.

  1. Client - computer and browser. Like the one I'm typing on.
  2. The Internet> - Nuff said.
  3. Servers - big computers that store whole bunch of info.
    • Here is a good video about servers.
HTML:

HTML stands for Hypertext Markup Language. All web pages are built on HTML and that's why it's important to learn.

Tag and Elements:

HTML documents are made up of these things called elements. Elements are written with a start tag, an end tag and content in between. Here is a link about elements.
And here are some of the tags we've talked about so far:

  • <div> - creates a box that can hold content and other tags.
  • <p> - stands for paragraph and is a block element that creates margins before and after each tag.

Computers are stupid:

There is a really cool way we explain why computers are stupid to kids. I heard this on a podcast once. Have a teacher pretend to be a computer that has to make a PB and J sandwhich. The teacher takes literal orders from the kids and makes a mess of things. The kids learn that telling a computer to make a sandwhich is a complex and detailed task.

Inline vs Block Elements:

HTML elements are either inline or block. The difference is that block elements have an "invisble box" around them used to format the element.

Lesson 2: Websites and HTML Boxes

How websites work:

Three main languages are used to make responsive websites. Think of it like building a house.

  1. HTML: which is like the structure of the house.
  2. CSS: which is like the style of the house (e.g. what color are the walls).
  3. JavaScript: which is like changing an element of the house (e.g. garage door opener)

DOM:

Document Object Model - a tree structure that is like the blue print for a webpage that can interact with elements in HTML.

  • Here is a link to a wiki-article on DOM.

Everything Is A Box:

HTML uses boxes for layout and structure. Boxes are also useful for translating a design into code. They also make it easier to rearrange the layout on a website. It’s a good idea once you get a mock-design to spec out how many boxes you want to use to design the page. Use <div> tags to create “boxes” in HTML.

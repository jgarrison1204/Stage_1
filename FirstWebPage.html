Lesson 1: The Basics of the Web

How the web works:

There are three major components that makes the web work.
  1. Client - computer and browser. Like the one I'm typing on.
  2. The Internet> - Nuff said.
  3. Servers - big computers that store whole bunch of info.
    • Here is a good video about servers.

HTML:

HTML stands for Hypertext Markup Language. All web pages are built on HTML and that's why it's important to learn.

Tag and Elements:

HTML documents are made up of these things called elements. Elements are written with a start tag, an end tag and content in between. Here is a link about elements.
And here are some of the tags we've talked about so far:

  • <div> - creates a box that can hold content and other tags.
  • <p> - stands for paragraph and is a block element that creates margins before and after each tag.

Computers are stupid:

There is a really cool way we explain why computers are stupid to kids. I heard this on a podcast once. Have a teacher pretend to be a computer that has to make a PB and J sandwhich. The teacher takes literal orders from the kids and makes a mess of things. The kids learn that telling a computer to make a sandwhich is a complex and detailed task.

Inline vs Block Elements:

HTML elements are either inline or block. The difference is that block elements have an "invisble box" around them used to format the element.

Lesson 2: Websites and HTML Boxes

How websites work:

Three main languages are used to make responsive websites. Think of it like building a house.

  • HTML: which is like the structure of the house.
  • CSS: which is like the style of the house (e.g. what color are the walls).
  • JavaScript: which is like changing an element of the house (e.g. garage door opener)

DOM:

Document Object Model - a tree structure that is like the blue print for a webpage that can interact with elements in HTML.

  • Here is a link to a wiki-article on DOM.

Everything Is A Box:

HTML uses boxes for layout and structure. Boxes are also useful for translating a design into code. They also make it easier to rearrange the layout on a website. It’s a good idea once you get a mock-design to spec out how many boxes you want to use to design the page. Use <div> tags to create “boxes” in HTML. Lesson 3: CSS Basics

Cascading Style Sheet (CSS) General Info

Starting a file:

Link a style sheet to an html file we put “link” in the head section and enter the correct path to the css file.CSS the most specific rule is applied to every element. (e.g. the rules you make for your house would be the same as national rule). This is what cascading refers to.

Steps:

  • First, writing a selector like an HTML tag.
  • If you wanted to apply a style to all paragraphs on a page you would do p { } and all paragraphs in the html layout would be styled.
  • Typically you use class as selector e.g. class=”test” then .test{ color: red;}
  • Or you can add a < style > inside < head > and write CSS in the < style> tag. We can also use < style > inside individual < div >. e.g. < div style="background-color: red; color: white" > Now, what if I change my mind? </div >

Displaying Divs Next To Each Other

For several elements to be next to each other change display value in the parent container to flex. See an example below:

.app{
display: -webkit-flex;
display: flex;
}

Dev Tools and Verification

Dev tools are a cool way to manipulate code in the browser to experiment with styling without having to write code in a text editor.
Verification allows you to make sure your HTML and CSS meets best practices in order to avoid throwing an error and preventing issues down the road.

The Box Model

The Box Model: HTML elements are considered boxes. The box model refers to the design and layout of a page. This CSS box model essentially wraps around HTML elements.

  • Content-The content of a box, where text and images appear.
  • Padding–Clears and area around the content. The padding is transparent.
  • Border–A border that goes around the padding and content.
  • Margin–space surrounding the border. The margin is transparent.
